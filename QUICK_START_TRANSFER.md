# 快速开始：导入私钥并转账

## 🚀 5分钟完成第一笔转账

### 步骤 1：获取私钥

打开终端，执行以下命令获取默认账户的私钥：

```bash
# 获取 qa 账户的私钥
/Users/ashy/go/bin/zethchaind keys export qa --unarmored-hex --unsafe -y --keyring-backend test
```

你会看到类似这样的输出（64位十六进制字符串）：
```
a1b2c3d4e5f6789012345678901234567890abcdef1234567890abcdef123456
```

**复制这个私钥！**

### 步骤 2：导入私钥到前端

1. **打开前端**
   - 访问 http://localhost:3000
   - 点击「账户管理」

2. **进入导入页面**
   - 点击「创建账户」标签
   - 点击「📥 导入私钥」按钮（右侧）

3. **填写信息**
   - **账户名称**：输入 `qa`（或任何你喜欢的名字）
   - **私钥**：粘贴步骤1获取的64位十六进制字符串
   - 点击「导入账户」

4. **等待成功提示**
   - 看到 "账户创建成功！" 提示
   - 自动返回账户列表

### 步骤 3：验证导入成功

回到「我的账户」列表，你应该看到：

```
✅ qa
   🔵 可转账  🟢 链上
   地址: zeth1gj9...
   ZETH 余额: 6900.000000 ZETH
```

**关键标识**：
- 🔵 **可转账** - 说明已成功导入私钥
- 🟢 **链上** - 说明账户在区块链上有余额

### 步骤 4：执行第一笔转账

1. **进入转账页面**
   - 点击顶部导航的「转账」

2. **选择发送账户**
   - 在「发送者」下拉框中选择刚导入的 `qa` 账户
   - 看到余额显示：6900.000000 ZETH

3. **填写转账信息**
   - **接收地址**：输入 qb 或 qc 的地址（可以从「我的账户」列表复制）
   - **代币类型**：选择 ZETH（默认）
   - **转账金额**：输入 `100`（转账 100 ZETH）
   - **备注**：可选，如 "测试转账"

4. **发送转账**
   - 点击「发送转账」按钮
   - 等待交易确认（约5-10秒）

5. **查看结果**
   - 看到 ✅ **转账成功！** 提示
   - 显示交易哈希和区块高度
   - qa 余额减少：6900 → 6799.995 ZETH（扣除100 ZETH + gas费）
   - qb 余额增加：7000 → 7100 ZETH

🎉 **恭喜！你已完成第一笔区块链转账！**

## 💡 常见场景

### 场景 1：只读账户升级为可转账账户

如果你已经有只读的 qa、qb、qc 账户（自动同步的）：

1. 获取对应账户的私钥
2. 使用「导入私钥」功能
3. 系统会自动将只读账户升级为可转账账户
4. 不会创建重复账户！

**识别方法**：
- 导入前：🟡 **只读** 标签
- 导入后：🔵 **可转账** 标签

### 场景 2：导入多个默认账户

按顺序导入所有三个默认账户：

```bash
# 1. 导出 qa 私钥
/Users/ashy/go/bin/zethchaind keys export qa --unarmored-hex --unsafe -y --keyring-backend test
# 在前端导入 → qa (可转账)

# 2. 导出 qb 私钥
/Users/ashy/go/bin/zethchaind keys export qb --unarmored-hex --unsafe -y --keyring-backend test
# 在前端导入 → qb (可转账)

# 3. 导出 qc 私钥
/Users/ashy/go/bin/zethchaind keys export qc --unarmored-hex --unsafe -y --keyring-backend test
# 在前端导入 → qc (可转账)
```

现在你有三个可转账的账户，可以互相转账测试！

### 场景 3：批量转账测试

导入所有三个账户后，你可以：

1. **qa → qb**: 转账 100 ZETH
2. **qb → qc**: 转账 50 ZETH
3. **qc → qa**: 转账 75 ZETH

每笔交易都会被记录在区块链上，可以在「链上账户」标签查看交易历史。

## ❓ 常见问题

### Q: 导入私钥后，为什么还有两个相同地址的账户？

**A**: 一个是只读账户（自动同步），一个是可转账账户（导入私钥）。

**解决方法**：删除只读账户，只保留可转账账户。

### Q: 导入时提示"该账户已存在且已有私钥"

**A**: 说明你已经导入过这个私钥了，无需重复导入。直接使用该账户转账即可。

### Q: 转账时提示"该账户没有私钥"

**A**: 你选择的是只读账户。请选择带有 🔵 **可转账** 标签的账户。

### Q: 私钥会保存在哪里？

**A**: 浏览器的 IndexedDB 中（本地存储），**以明文形式保存**。

⚠️ **安全提醒**：
- 仅用于开发和学习
- 不要用于生产环境
- 不要存储真实资产
- 定期清理测试私钥

### Q: 如何删除已导入的私钥？

**A**: 在「我的账户」列表中，找到该账户，点击删除按钮。或者清空浏览器 IndexedDB：
```
F12 → Application → IndexedDB → ZETHWallet → accounts → Clear
```
